Chetana shell — bundle report

This file is regenerated every time you run pnpm --filter @chetana/shell run analyze. The committed copy is a placeholder so the path web/apps/shell/bundle-report.html exists in git per TASK-P1-WEB-002 acceptance #3.

Run pnpm install followed by pnpm --filter @chetana/shell run analyze from the repo root (or cd web && pnpm install first if you haven't yet) to overwrite this file with the real treemap visualisation.

What the report verifies

  • The initial JS bundle (the entrypoint loaded before any route navigation) does not contain @cesium/engine — the Cesium use sites route through src/lib/cesium/loader.ts which dynamic-imports the package, and vite.config.ts's manualChunks function splits the chunk into a stably-named cesium-engine output.
  • The shell entrypoint stays under 200 KB gzip (a soft guard the bundle-analyser report makes easy to inspect by sorting chunks by gzip size).
