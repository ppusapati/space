## 2. E-GS-001: Satellite Catalog

> **Epic Goal:** Manage satellite registration and TLE data for orbit prediction

### US-GS-001: Register Satellite

**Epic:** E-GS-001  
**Use Case:** UC-GS-001  
**Priority:** Must Have  
**Story Points:** 5

**As a** mission operator  
**I want to** register a satellite in the system  
**So that** I can schedule passes and track it

**Acceptance Criteria:**

```gherkin
Scenario: Register new satellite
  Given I am an authenticated operator
  When I create a satellite with:
    | Field           | Value                |
    | name            | P9E-SAT-001          |
    | norad_id        | 55001                |
    | international_id| 2024-001A            |
    | description     | Earth observation    |
  Then the satellite is registered
  And status is set to "active"
  And I can configure its parameters

Scenario: Duplicate NORAD ID
  Given a satellite with NORAD ID 55001 exists
  When I try to register another with same NORAD ID
  Then I see "Satellite with this NORAD ID already exists"

Scenario: Quota check
  Given the tenant has reached satellite quota
  When I try to register a new satellite
  Then I see "Satellite limit reached"
```

**Technical Notes:**
- NORAD ID must be unique within tenant
- International designator format: YYYY-NNNL
- Auto-fetch initial TLE after registration
- Create default telemetry parameter set

**Dependencies:** E-PLT-002 (Tenant)

- Archive previous TLEs for history
- Trigger pass re-prediction after update

### US-GS-006: Automatic TLE Refresh

**Epic:** E-GS-001  
**Use Case:** UC-GS-002  
**Priority:** Must Have  
**Story Points:** 5

**As the** system  
**I want to** automatically refresh TLEs from Space-Track  
**So that** orbit predictions remain accurate

**Acceptance Criteria:**

```gherkin
Scenario: Scheduled TLE refresh
  Given the TLE refresh job is scheduled
  When the job runs every 6 hours
  Then it queries Space-Track for all registered NORAD IDs
  And updates TLEs that have changed
  And logs refresh results

Scenario: TLE updated
  Given a satellite's TLE has changed on Space-Track
  When the refresh job runs
  Then the new TLE is saved
  And pass predictions are recalculated
  And operators are notified of significant changes

Scenario: Space-Track rate limit
  Given Space-Track returns HTTP 429
  When the refresh job encounters this
  Then it backs off exponentially
  And retries after the Retry-After period

Scenario: Stale TLE alert
  Given a satellite's TLE is older than 48 hours
  And no update is available
  Then an alert is raised
  And operators are notified
```

**Technical Notes:**
- Space-Track API credentials per tenant or system-wide
- Query in batches (max 20 NORAD IDs per request)
- Respect rate limits (4 requests per minute)
- Compare TLE epoch to detect changes

**Dependencies:** US-GS-005, E-CMN-003 (Scheduler)
