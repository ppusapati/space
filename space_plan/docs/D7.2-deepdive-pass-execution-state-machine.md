## 1. Executive Summary

### 1.1 Purpose

The Pass Execution State Machine (PESM) is the core controller for satellite pass operations. It manages the complete lifecycle of a satellite pass from initial scheduling through execution completion, ensuring:

- Deterministic state transitions
- Proper resource allocation and release
- Coordinated equipment control
- Reliable telemetry/command operations
- Comprehensive audit logging
- Graceful error handling and recovery

### 1.2 Design Goals

| Goal | Description |
|------|-------------|
| **Determinism** | Every state transition is explicit and predictable |
| **Reliability** | No pass left in undefined state; always recoverable |
| **Observability** | Complete audit trail of all state changes |
| **Extensibility** | Easy to add new states or transition logic |
| **Concurrency** | Handle multiple simultaneous passes across stations |
| **Fault Tolerance** | Graceful degradation on equipment failures |

### 1.3 Key Concepts

| Concept | Definition |
|---------|------------|
| **Pass** | A scheduled satellite contact opportunity |
| **State** | Current phase of pass execution |
| **Event** | Trigger that may cause state transition |
| **Guard** | Condition that must be true for transition |
| **Action** | Side effect executed during transition |
| **Timeout** | Time-based automatic transition trigger |

### 2.2 State Categories

| Category | States | Description |
|----------|--------|-------------|
| **Initial** | SCHEDULED | Pass awaiting execution |
| **Pre-Pass** | PREPARING, READY, PREP_FAILED | Equipment preparation phase |
| **Active** | ACQUIRING, TRACKING, CLOSING | Pass in progress |
| **Post-Pass** | REPORTING | Generating pass report |
| **Terminal** | COMPLETED, FAILED, CANCELLED, ABORTED | Final states |

### 2.3 State Flow Summary

```
Normal Flow:
SCHEDULED → PREPARING → READY → ACQUIRING → TRACKING → CLOSING → REPORTING → COMPLETED

Cancel Flow:
Any non-terminal state → CANCELLED

Failure Flow:
PREPARING → PREP_FAILED → REPORTING → COMPLETED (partial)
ACQUIRING/TRACKING → REPORTING → COMPLETED (partial)

Abort Flow (emergency):
Any active state → ABORTED
```

## 3. State Definitions

### 3.1 SCHEDULED

**Description:** Pass has been created and is awaiting execution.

```yaml
state: SCHEDULED
category: initial
entry_actions:
  - allocate_pass_resources
  - schedule_preparation_timer
  - notify_operators
invariants:
  - pass.aos_time > current_time
  - station.is_available(pass.aos_time, pass.los_time)
allowed_events:
  - PREPARATION_TIME_REACHED
  - CANCEL_REQUESTED
  - MODIFY_REQUESTED
exit_actions:
  - cancel_preparation_timer
```

**Entry Conditions:**
- Pass successfully scheduled
- No conflicts with other passes
- Equipment available

**Data Requirements:**
| Field | Required | Description |
|-------|----------|-------------|
| satellite_id | Yes | Target satellite |
| station_id | Yes | Ground station |
| aos_time | Yes | Acquisition of Signal time |
| los_time | Yes | Loss of Signal time |
| max_elevation | Yes | Peak elevation angle |
| priority | Yes | Scheduling priority |
| operations | No | Planned operations list |

---

### 3.2 PREPARING

**Description:** Pre-pass preparation in progress. Equipment being configured.

```yaml
state: PREPARING
category: pre_pass
entry_actions:
  - start_preparation_sequence
  - configure_antenna
  - configure_radio
  - load_satellite_configuration
  - verify_tle_current
timeout:
  duration: 300s  # 5 minutes
  on_timeout: PREP_FAILED
invariants:
  - time_to_aos > 0
  - station.equipment_responsive
allowed_events:
  - PREPARATION_COMPLETE
  - PREPARATION_FAILED
  - CANCEL_REQUESTED
  - EQUIPMENT_FAULT
exit_actions:
  - stop_preparation_sequence
```

**Preparation Sequence:**
1. Verify TLE data is current (< 24h old)
2. Calculate initial pointing angles
3. Command antenna to pre-position
4. Configure radio frequency and mode
5. Initialize telemetry decoders
6. Verify command uplink ready (if applicable)
7. Run system self-test

**Typical Duration:** 2-5 minutes before AOS

---

### 3.3 READY

**Description:** Preparation complete. Awaiting AOS.

```yaml
state: READY
category: pre_pass
entry_actions:
  - arm_aos_detection
  - start_tracking_prediction
  - notify_ready
invariants:
  - equipment.fully_configured
  - antenna.pre_positioned
  - time_to_aos <= preparation_lead_time
allowed_events:
  - AOS_DETECTED
  - AOS_TIMEOUT
  - CANCEL_REQUESTED
  - EQUIPMENT_FAULT
exit_actions:
  - log_ready_duration
```

**Conditions for Entry:**
- All equipment configured successfully
- Antenna pointed to expected AOS azimuth/elevation
- Radio locked on expected frequency
- TLE verified current

---

### 3.4 PREP_FAILED

**Description:** Preparation sequence failed. Pass may be recoverable.

```yaml
state: PREP_FAILED
category: pre_pass
entry_actions:
  - log_failure_details
  - attempt_recovery
  - notify_operators
allowed_events:
  - RETRY_PREPARATION
  - CANCEL_REQUESTED
  - SKIP_TO_REPORTING
exit_actions:
  - record_failure_metrics
```

**Recovery Options:**
| Failure Type | Recovery Action |
|--------------|-----------------|
| Antenna timeout | Retry command, check limits |
| Radio config failed | Reset radio, retry |
| TLE stale | Fetch fresh TLE, recalculate |
| Self-test failed | Identify component, attempt bypass |

---

### 3.5 ACQUIRING

**Description:** AOS reached. Attempting to acquire satellite signal.

```yaml
state: ACQUIRING
category: active
entry_actions:
  - start_signal_search
  - begin_antenna_tracking
  - start_telemetry_recording
  - start_aos_timer
timeout:
  duration: 60s
  on_timeout: transition_to_TRACKING_or_FAILED
invariants:
  - current_time >= aos_time
  - antenna.tracking_active
allowed_events:
  - SIGNAL_LOCKED
  - ACQUISITION_TIMEOUT
  - CANCEL_REQUESTED
  - EQUIPMENT_FAULT
  - ABORT_REQUESTED
exit_actions:
  - log_acquisition_result
```

**Signal Acquisition Process:**
1. Enable AGC (Automatic Gain Control)
2. Search for carrier signal
3. Achieve frequency lock
4. Achieve frame sync
5. Begin telemetry decode

**Acquisition Metrics:**
- Time to first frame
- Signal strength (RSSI)
- Frequency offset
- Bit error rate estimate

---

### 3.6 TRACKING

**Description:** Satellite locked. Active pass operations in progress.

```yaml
state: TRACKING
category: active
entry_actions:
  - confirm_signal_lock
  - start_operations_sequence
  - enable_command_uplink
  - start_tracking_metrics
sub_states:
  - TRACKING_NOMINAL
  - TRACKING_DEGRADED
  - TRACKING_COMMANDING
invariants:
  - signal_locked OR time_since_lock_loss < reacquisition_timeout
  - current_time < los_time
allowed_events:
  - LOS_DETECTED
  - LOS_PREDICTED
  - SIGNAL_LOST
  - SIGNAL_REACQUIRED
  - COMMAND_REQUESTED
  - COMMAND_COMPLETED
  - CANCEL_REQUESTED
  - ABORT_REQUESTED
  - EQUIPMENT_FAULT
exit_actions:
  - disable_command_uplink
  - stop_operations_sequence
  - finalize_tracking_metrics
```

**Sub-States:**

```
┌─────────────────────────────────────────────────────────────────┐
│                    TRACKING SUB-STATES                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐      signal_lost     ┌──────────────┐        │
│  │   NOMINAL    │─────────────────────▶│   DEGRADED   │        │
│  │              │◀─────────────────────│              │        │
│  └──────┬───────┘    signal_reacquired └──────────────┘        │
│         │                                                       │
│         │ command_start                                         │
│         ▼                                                       │
│  ┌──────────────┐                                              │
│  │  COMMANDING  │ (while command in progress)                  │
│  └──────────────┘                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.7 CLOSING

**Description:** LOS approaching or reached. Finalizing pass operations.

```yaml
state: CLOSING
category: active
entry_actions:
  - complete_pending_commands
  - flush_telemetry_buffers
  - start_antenna_stow_sequence
timeout:
  duration: 120s
  on_timeout: force_transition_to_REPORTING
invariants:
  - current_time >= los_time OR los_approaching
allowed_events:
  - CLOSE_COMPLETE
  - CLOSE_TIMEOUT
  - EQUIPMENT_FAULT
exit_actions:
  - record_los_metrics
  - release_equipment
```

**Closing Sequence:**
1. Complete any in-flight commands
2. Send final telemetry request (if applicable)
3. Flush all telemetry buffers to storage
4. Stop tracking updates
5. Begin antenna stow/park
6. Release radio resources

---

### 3.8 REPORTING

**Description:** Generating pass execution report.

```yaml
state: REPORTING
category: post_pass
entry_actions:
  - aggregate_pass_metrics
  - generate_pass_report
  - calculate_statistics
  - archive_raw_data
timeout:
  duration: 300s
  on_timeout: complete_with_partial_report
allowed_events:
  - REPORT_COMPLETE
  - REPORT_TIMEOUT
exit_actions:
  - publish_report
  - notify_stakeholders
```

**Report Contents:**
- Pass summary (times, duration, max elevation)
- Telemetry statistics (frames received, errors, gaps)
- Command execution results
- Signal quality metrics
- Anomalies and alerts
- Equipment performance

### 3.9 Terminal States

#### COMPLETED

```yaml
state: COMPLETED
category: terminal
entry_actions:
  - mark_pass_complete
  - release_all_resources
  - update_statistics
  - archive_pass_record
result: success | partial_success | degraded
```

**Completion Categories:**
| Result | Description |
|--------|-------------|
| SUCCESS | All objectives achieved |
| PARTIAL_SUCCESS | Some objectives achieved |
| DEGRADED | Completed with issues |

#### FAILED

```yaml
state: FAILED
category: terminal
entry_actions:
  - mark_pass_failed
  - release_all_resources
  - log_failure_analysis
  - trigger_incident_if_needed
failure_reason: required
```

**Failure Reasons:**
- EQUIPMENT_FAILURE
- NO_SIGNAL_ACQUIRED
- CONFIGURATION_ERROR
- EXTERNAL_INTERFERENCE
- SATELLITE_ANOMALY

#### CANCELLED

```yaml
state: CANCELLED
category: terminal
entry_actions:
  - mark_pass_cancelled
  - release_all_resources
  - notify_stakeholders
cancellation_reason: required
cancelled_by: required
```

#### ABORTED

```yaml
state: ABORTED
category: terminal
entry_actions:
  - emergency_stop_all
  - safe_equipment
  - release_all_resources
  - trigger_incident
abort_reason: required
```

**Abort Triggers:**
- Emergency stop button
- Safety interlock triggered
- Critical equipment failure
- Operator emergency command

### 4.2 Transition Definition Schema

```go
// transition.go

package passexec

// Transition defines a valid state transition
type Transition struct {
    FromState   State
    ToState     State
    Event       Event
    Guard       GuardFunc
    Actions     []ActionFunc
    Description string
}

// GuardFunc returns true if transition is allowed
type GuardFunc func(ctx *PassContext) bool

// ActionFunc performs side effect during transition
type ActionFunc func(ctx *PassContext) error

// PassContext contains all pass execution context
type PassContext struct {
    Pass          *Pass
    Station       *GroundStation
    Satellite     *Satellite
    Equipment     *EquipmentState
    Telemetry     *TelemetryState
    Commands      *CommandQueue
    Metrics       *PassMetrics
    CurrentTime   time.Time
}
```

### 4.3 Transition Definitions

```go
// transitions.go

var transitions = []Transition{
    // SCHEDULED → PREPARING
    {
        FromState:   StateScheduled,
        ToState:     StatePreparing,
        Event:       EventPrepTimeReached,
        Guard:       guardTimeToAOS,
        Actions:     []ActionFunc{actionStartPreparation},
        Description: "Begin pass preparation sequence",
    },
    
    // SCHEDULED → CANCELLED
    {
        FromState:   StateScheduled,
        ToState:     StateCancelled,
        Event:       EventCancelRequested,
        Guard:       guardAuthorizedCancel,
        Actions:     []ActionFunc{actionRecordCancellation},
        Description: "Cancel scheduled pass",
    },
    
    // PREPARING → READY
    {
        FromState:   StatePreparing,
        ToState:     StateReady,
        Event:       EventPrepComplete,
        Guard:       guardAllChecksPassed,
        Actions:     []ActionFunc{actionArmAOSDetection},
        Description: "Preparation complete, ready for AOS",
    },
    
    // PREPARING → PREP_FAILED
    {
        FromState:   StatePreparing,
        ToState:     StatePrepFailed,
        Event:       EventPrepFailed,
        Guard:       nil, // Always allowed
        Actions:     []ActionFunc{actionLogPrepFailure, actionAttemptRecovery},
        Description: "Preparation sequence failed",
    },
    
    // READY → ACQUIRING
    {
        FromState:   StateReady,
        ToState:     StateAcquiring,
        Event:       EventAOSDetected,
        Guard:       guardSignalPresent,
        Actions:     []ActionFunc{actionStartAcquisition, actionStartRecording},
        Description: "AOS detected, begin signal acquisition",
    },
    
    // ACQUIRING → TRACKING
    {
        FromState:   StateAcquiring,
        ToState:     StateTracking,
        Event:       EventSignalLocked,
        Guard:       guardFrameSync,
        Actions:     []ActionFunc{actionConfirmLock, actionStartOperations},
        Description: "Signal locked, begin tracking",
    },
    
    // TRACKING → CLOSING
    {
        FromState:   StateTracking,
        ToState:     StateClosing,
        Event:       EventLOSDetected,
        Guard:       nil,
        Actions:     []ActionFunc{actionBeginClosing},
        Description: "LOS reached, begin close sequence",
    },
    
    // CLOSING → REPORTING
    {
        FromState:   StateClosing,
        ToState:     StateReporting,
        Event:       EventCloseComplete,
        Guard:       nil,
        Actions:     []ActionFunc{actionGenerateReport},
        Description: "Close complete, generate report",
    },
    
    // REPORTING → COMPLETED
    {
        FromState:   StateReporting,
        ToState:     StateCompleted,
        Event:       EventReportComplete,
        Guard:       nil,
        Actions:     []ActionFunc{actionFinalizePass, actionReleaseResources},
        Description: "Report complete, pass finished",
    },
    
    // Any active state → ABORTED
    {
        FromState:   StateAny, // Special: matches ACQUIRING, TRACKING, CLOSING
        ToState:     StateAborted,
        Event:       EventAbortRequested,
        Guard:       nil,
        Actions:     []ActionFunc{actionEmergencyStop, actionSafeEquipment},
        Description: "Emergency abort",
    },
}
```

## 5. Events and Triggers

### 5.1 Event Categories

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           EVENT CATEGORIES                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  TIMER EVENTS (Automatic)                                                  │
│  ═══════════════════════                                                   │
│  • PREP_TIME_REACHED     - Preparation start time                         │
│  • AOS_TIME_REACHED      - Predicted AOS time                             │
│  • LOS_PREDICTED         - Approaching predicted LOS                       │
│  • TIMEOUT_EXPIRED       - State-specific timeout                         │
│                                                                             │
│  EQUIPMENT EVENTS (From Hardware)                                          │
│  ═════════════════════════════════                                         │
│  • SIGNAL_DETECTED       - RF signal present                              │
│  • SIGNAL_LOCKED         - Carrier and frame lock achieved                │
│  • SIGNAL_LOST           - Lock lost                                       │
│  • SIGNAL_REACQUIRED     - Lock regained                                  │
│  • EQUIPMENT_FAULT       - Hardware error                                  │
│  • EQUIPMENT_READY       - Hardware configured                            │
│                                                                             │
│  OPERATOR EVENTS (From UI/API)                                             │
```

```go
// guards.go

package passexec

// guardTimeToAOS checks if preparation should start
func guardTimeToAOS(ctx *PassContext) bool {
    leadTime := ctx.Station.PreparationLeadTime
    timeToAOS := ctx.Pass.AOSTime.Sub(ctx.CurrentTime)
    return timeToAOS <= leadTime && timeToAOS > 0
}

// guardAuthorizedCancel checks if user can cancel
func guardAuthorizedCancel(ctx *PassContext) bool {
    // Check user has permission
    if !ctx.User.HasPermission("pass:cancel") {
        return false
    }
    
    // Check pass is not in critical phase
    if ctx.Pass.State == StateTracking && ctx.Pass.HasActiveCommands() {
        // Require elevated permission during active commanding
        return ctx.User.HasPermission("pass:cancel:active")
    }
    
    return true
}

// guardAllChecksPassed verifies preparation complete
func guardAllChecksPassed(ctx *PassContext) bool {
    checks := ctx.PreparationChecks
    
    return checks.AntennaPositioned &&
           checks.RadioConfigured &&
           checks.TLEVerified &&
           checks.DecodersReady &&
           checks.SelfTestPassed
}

// guardSignalPresent checks for RF signal
func guardSignalPresent(ctx *PassContext) bool {
    return ctx.Equipment.Radio.RSSI > ctx.Station.MinSignalThreshold
}

// guardFrameSync checks for valid telemetry frames
func guardFrameSync(ctx *PassContext) bool {
    return ctx.Equipment.Decoder.FrameSync &&
           ctx.Equipment.Decoder.ValidFrameCount > 0
}

// guardRetriesRemaining checks if retry allowed
func guardRetriesRemaining(ctx *PassContext) bool {
    return ctx.Pass.PreparationRetries < ctx.Station.MaxPreparationRetries
}

// guardLOSApproaching checks if near end of pass
func guardLOSApproaching(ctx *PassContext) bool {
    timeToLOS := ctx.Pass.LOSTime.Sub(ctx.CurrentTime)
    return timeToLOS <= 30*time.Second
}

// guardNoActiveCommands checks command queue empty
func guardNoActiveCommands(ctx *PassContext) bool {
    return ctx.Commands.PendingCount() == 0 &&
           ctx.Commands.InFlightCount() == 0
}
```

```go
    log := ctx.Logger.With("pass_id", ctx.Pass.ID)

    // Record lock time
    ctx.Metrics.LockAcquiredTime = ctx.CurrentTime
    ctx.Metrics.TimeToLock = ctx.CurrentTime.Sub(ctx.Metrics.AcquisitionStartTime)

    if ctx.Pass.HasCommandOperations() {

    for _, op := range ctx.Pass.Operations {

            ctx.OperationQueue.Enqueue(op)
        }
    }
    
    return nil

    return nil
}
```

```go
// timeout_manager.go

package passexec

import (
    "context"
    "sync"
    "time"
)

// TimeoutManager handles state-specific timeouts
type TimeoutManager struct {
    timers   map[string]*time.Timer
    mu       sync.Mutex
    executor *StateMachineExecutor
    config   *TimeoutConfig
}

// TimeoutEntry represents an active timeout
type TimeoutEntry struct {
    PassID    string
    State     State
    StartTime time.Time
    Duration  time.Duration
    Timer     *time.Timer
    Callback  func()
}

// StartTimeout begins a timeout for a pass state
func (tm *TimeoutManager) StartTimeout(passID string, state State) {
    tm.mu.Lock()
    defer tm.mu.Unlock()
    
    // Cancel any existing timeout for this pass
    key := fmt.Sprintf("%s:%s", passID, state)
    if existing, ok := tm.timers[key]; ok {
        existing.Stop()
    }
    
    // Get timeout duration for this state
    duration := tm.config.GetDuration(state)
    if duration == 0 {
        return // No timeout for this state
    }
    
    // Create new timer
    timer := time.AfterFunc(duration, func() {
        tm.handleTimeout(passID, state)
    })
    
    tm.timers[key] = timer
}

// CancelTimeout stops an active timeout
func (tm *TimeoutManager) CancelTimeout(passID string, state State) {
    tm.mu.Lock()
    defer tm.mu.Unlock()
    
    key := fmt.Sprintf("%s:%s", passID, state)
    if timer, ok := tm.timers[key]; ok {
        timer.Stop()
        delete(tm.timers, key)
    }
}

// handleTimeout processes a timeout expiration
func (tm *TimeoutManager) handleTimeout(passID string, state State) {
    log := tm.logger.With("pass_id", passID, "state", state)
    log.Warn("State timeout expired")
    
    // Get pass context
    ctx, err := tm.executor.GetPassContext(passID)
    if err != nil {
        log.Error("Failed to get pass context", "error", err)
        return
    }
    
    // Determine timeout action based on state
    switch state {
    case StatePreparing:
        tm.handlePreparationTimeout(ctx)
        
    case StateReady:
        tm.handleReadyTimeout(ctx)
        
    case StateAcquiring:
        tm.handleAcquisitionTimeout(ctx)
        
    case StateTracking:
        tm.handleTrackingTimeout(ctx)
        
    case StateClosing:
        tm.handleClosingTimeout(ctx)
        
    case StateReporting:
        tm.handleReportingTimeout(ctx)
    }
}

// handleAcquisitionTimeout processes acquisition timeout
func (tm *TimeoutManager) handleAcquisitionTimeout(ctx *PassContext) {
    // Check if we're approaching LOS
    timeToLOS := ctx.Pass.LOSTime.Sub(ctx.CurrentTime)
    
    if timeToLOS < 60*time.Second {
        // Near LOS, transition to CLOSING
        tm.executor.ProcessEvent(ctx.Pass.ID, EventPayload{
            Event:  EventTimeoutExpired,
            Reason: "Acquisition timeout near LOS",
            Data:   map[string]interface{}{"target_state": StateClosing},
        })
    } else {
        // Far from LOS, mark as failed
        tm.executor.ProcessEvent(ctx.Pass.ID, EventPayload{
            Event:  EventTimeoutExpired,
            Reason: "Acquisition timeout - no signal acquired",
            Data:   map[string]interface{}{"target_state": StateFailed},
        })
    }
}

// handleTrackingTimeout processes signal loss timeout
func (tm *TimeoutManager) handleTrackingTimeout(ctx *PassContext) {
    // Signal lost for too long, check LOS proximity
    timeToLOS := ctx.Pass.LOSTime.Sub(ctx.CurrentTime)
    
    if timeToLOS < 60*time.Second {
        // Near LOS anyway, close normally
        tm.executor.ProcessEvent(ctx.Pass.ID, EventPayload{
            Event:  EventLOSDetected,
            Reason: "Signal loss near predicted LOS",
        })
    } else {
        // Unexpected signal loss
        ctx.Metrics.Anomalies = append(ctx.Metrics.Anomalies, Anomaly{
            Time:        ctx.CurrentTime,
            Type:        AnomalySignalLoss,
            Description: "Unexpected signal loss during tracking",
        })
        
        tm.executor.ProcessEvent(ctx.Pass.ID, EventPayload{
            Event:  EventSignalLost,
            Reason: "Signal reacquisition timeout",
        })
    }
}
```

```go
package passexec

// RecoveryStrategy defines how to handle specific errors
type RecoveryStrategy struct {
    ErrorType     ErrorType
    MaxRetries    int
    RetryDelay    time.Duration
    RecoveryFunc  func(ctx *PassContext, err error) error
    FallbackFunc  func(ctx *PassContext, err error) error
    EscalateAfter int // Escalate to operator after N failures
}

var recoveryStrategies = map[ErrorType]RecoveryStrategy{
    ErrorAntennaTimeout: {
        ErrorType:     ErrorAntennaTimeout,
        MaxRetries:    3,
        RetryDelay:    5 * time.Second,
        RecoveryFunc:  recoverAntennaTimeout,
        FallbackFunc:  fallbackAntennaFailure,
        EscalateAfter: 2,
    },
    ErrorSignalLost: {
        ErrorType:     ErrorSignalLost,
        MaxRetries:    0, // Continuous reacquisition attempts
        RetryDelay:    0,
        RecoveryFunc:  recoverSignalLoss,
        FallbackFunc:  fallbackSignalLoss,
        EscalateAfter: 1,
    },
    ErrorCommandTimeout: {
        ErrorType:     ErrorCommandTimeout,
        MaxRetries:    2,
        RetryDelay:    10 * time.Second,
        RecoveryFunc:  recoverCommandTimeout,
        FallbackFunc:  fallbackCommandFailure,
        EscalateAfter: 1,
    },
    ErrorUplinkFailure: {
        ErrorType:     ErrorUplinkFailure,
        MaxRetries:    1,
        RetryDelay:    30 * time.Second,
        RecoveryFunc:  recoverUplinkFailure,
        FallbackFunc:  fallbackContinueReceiveOnly,
        EscalateAfter: 1,
    },
}

// recoverAntennaTimeout attempts antenna recovery
func recoverAntennaTimeout(ctx *PassContext, err error) error {
    log := ctx.Logger.With("pass_id", ctx.Pass.ID)
    log.Warn("Attempting antenna recovery", "error", err)
    
    // Check antenna status
    status, err := ctx.Equipment.Antenna.GetStatus()
    if err != nil {
        return fmt.Errorf("antenna status check failed: %w", err)
    }
    
    // If antenna is in fault state, reset
    if status.InFault {
        if err := ctx.Equipment.Antenna.Reset(); err != nil {
            return fmt.Errorf("antenna reset failed: %w", err)
        }
        time.Sleep(5 * time.Second)
    }
    
    // Retry positioning
    azEl, _ := ctx.Propagator.PredictPosition(ctx.Satellite.TLE, ctx.Pass.AOSTime)
    return ctx.Equipment.Antenna.GoTo(azEl.Azimuth, azEl.Elevation)
}

// recoverSignalLoss handles temporary signal loss
func recoverSignalLoss(ctx *PassContext, err error) error {
    log := ctx.Logger.With("pass_id", ctx.Pass.ID)
    log.Warn("Signal lost, attempting reacquisition")
    
    // Update tracking to predicted position
    currentPos, _ := ctx.Propagator.PredictPosition(
        ctx.Satellite.TLE, 
        ctx.CurrentTime,
    )
    ctx.Equipment.Antenna.GoTo(currentPos.Azimuth, currentPos.Elevation)
    
    // Widen acquisition search
    ctx.Equipment.Radio.WidenSearch()
    
    // Set reacquisition timeout
    ctx.TimeoutManager.StartTimeout(ctx.Pass.ID, StateTrackingDegraded)
    
    // Update metrics
    ctx.Metrics.SignalDropouts++
    ctx.Metrics.LastSignalLoss = ctx.CurrentTime
    
    return nil // Not really an error to return, we're in degraded state
}

// fallbackContinueReceiveOnly continues without command capability
func fallbackContinueReceiveOnly(ctx *PassContext, err error) error {
    log := ctx.Logger.With("pass_id", ctx.Pass.ID)
    log.Warn("Continuing pass in receive-only mode", "error", err)
    
    // Disable uplink flag
    ctx.Pass.UplinkEnabled = false
    
    // Cancel any pending commands
    ctx.Commands.CancelAll("Uplink failure - receive only mode")
    
    // Notify operators
    ctx.Notifier.NotifyOperators(&OperatorAlert{
        PassID:   ctx.Pass.ID,
        Severity: SeverityWarning,
        Message:  "Pass continuing in receive-only mode due to uplink failure",
        Time:     ctx.CurrentTime,
    })
    
    // Record anomaly
    ctx.Metrics.Anomalies = append(ctx.Metrics.Anomalies, Anomaly{
        Time:        ctx.CurrentTime,
        Type:        AnomalyUplinkFailure,
        Description: "Uplink failure, continuing receive-only",
    })
    
    return nil
}
```

### 9.3 Error State Handling

```go
// error_handler.go

package passexec

// ErrorHandler processes errors during pass execution
type ErrorHandler struct {
    strategies map[ErrorType]RecoveryStrategy
    executor   *StateMachineExecutor
    metrics    *ErrorMetrics
}

// HandleError processes an error and determines action
func (eh *ErrorHandler) HandleError(ctx *PassContext, err error) error {
    errorType := classifyError(err)
    strategy, ok := eh.strategies[errorType]
    
    if !ok {
        // Unknown error type - log and continue if possible
        ctx.Logger.Error("Unclassified error", "error", err)
        return err
    }
    
    // Track retry count
    retryKey := fmt.Sprintf("%s:%s", ctx.Pass.ID, errorType)
    retryCount := eh.getRetryCount(retryKey)
    
    // Check if should escalate to operator
    if retryCount >= strategy.EscalateAfter {
        eh.escalateToOperator(ctx, err, retryCount)
    }
    
    // Attempt recovery if retries remaining
    if strategy.MaxRetries == 0 || retryCount < strategy.MaxRetries {
        eh.incrementRetryCount(retryKey)
        
        if strategy.RetryDelay > 0 {
            time.Sleep(strategy.RetryDelay)
        }
        
        if err := strategy.RecoveryFunc(ctx, err); err == nil {
            eh.resetRetryCount(retryKey)
            return nil // Recovery successful
        }
    }
    
    // Recovery failed or exhausted, use fallback
    if strategy.FallbackFunc != nil {
        return strategy.FallbackFunc(ctx, err)
    }
    
    // No fallback, error is fatal for this pass
    return fmt.Errorf("unrecoverable error: %w", err)
}

// escalateToOperator sends alert to operators
func (eh *ErrorHandler) escalateToOperator(ctx *PassContext, err error, retryCount int) {
    alert := &OperatorAlert{
        PassID:     ctx.Pass.ID,
        Severity:   SeverityWarning,
        Message:    fmt.Sprintf("Repeated error during pass: %v (attempt %d)", err, retryCount),
        Time:       ctx.CurrentTime,
        RequiresAck: true,
    }
    
    ctx.Notifier.NotifyOperators(alert)
}

// classifyError determines the error type
func classifyError(err error) ErrorType {
    switch {
    case errors.Is(err, ErrAntennaTimeout):
        return ErrorAntennaTimeout
    case errors.Is(err, ErrRadioConfig):
        return ErrorRadioConfig
    case errors.Is(err, ErrSignalLost):
        return ErrorSignalLost
    case errors.Is(err, ErrCommandTimeout):
        return ErrorCommandTimeout
    case errors.Is(err, ErrUplinkFailure):
        return ErrorUplinkFailure
    case errors.Is(err, ErrDecoderFailure):
        return ErrorDecoderFailure
    default:
        return ErrorUnknown
    }
}
```

```go
// executor.go

package passexec

import (
    "context"
    "sync"
    "time"
)

// PassExecutor manages execution of a single pass
type PassExecutor struct {
    pass           *Pass
    context        *PassContext
    stateMachine   *StateMachine
    timeoutManager *TimeoutManager
    errorHandler   *ErrorHandler
    eventChan      chan EventPayload
    mu             sync.RWMutex
    logger         *slog.Logger
}

// StateMachine implements the pass state machine
type StateMachine struct {
    transitions []Transition
    current     State
    mu          sync.RWMutex
    history     []StateChange
}

// StateChange records a state transition
type StateChange struct {
    FromState   State
    ToState     State
    Event       Event
    Timestamp   time.Time
    Duration    time.Duration
    TriggeredBy string
}

// NewPassExecutor creates a new executor for a pass
func NewPassExecutor(pass *Pass, deps *Dependencies) (*PassExecutor, error) {
    // Create pass context
    ctx := &PassContext{
        Pass:        pass,
        Station:     deps.GetStation(pass.StationID),
        Satellite:   deps.GetSatellite(pass.SatelliteID),
        Equipment:   deps.GetEquipment(pass.StationID),
        Telemetry:   deps.GetTelemetryAdapter(),
        Commands:    NewCommandQueue(),
        Metrics:     &PassMetrics{},
        CurrentTime: time.Now().UTC(),
    }
    
    // Initialize state machine
    sm := &StateMachine{
        transitions: buildTransitions(),
        current:     StateScheduled,
        history:     make([]StateChange, 0),
    }
    
    executor := &PassExecutor{
        pass:         pass,
        context:      ctx,
        stateMachine: sm,
        eventChan:    make(chan EventPayload, 100),
        logger:       deps.Logger.With("pass_id", pass.ID),
    }
    
    executor.timeoutManager = NewTimeoutManager(executor, deps.TimeoutConfig)
    executor.errorHandler = NewErrorHandler(recoveryStrategies, executor)
    
    return executor, nil
}

// Start begins pass execution
func (pe *PassExecutor) Start(ctx context.Context) error {
    pe.logger.Info("Starting pass executor")
    
    // Start event processing loop
    go pe.eventLoop(ctx)
    
    // Schedule preparation timer
    prepTime := pe.pass.AOSTime.Add(-pe.context.Station.PreparationLeadTime)
    pe.scheduleEvent(prepTime, EventPrepTimeReached)
    
    return nil
}

// eventLoop processes events and drives state transitions
func (pe *PassExecutor) eventLoop(ctx context.Context) {
    for {
        select {
        case <-ctx.Done():
            pe.logger.Info("Event loop terminated")
            return
            
        case event := <-pe.eventChan:
            pe.processEvent(event)
        }
    }
}

// processEvent handles an incoming event
func (pe *PassExecutor) processEvent(event EventPayload) {
    pe.mu.Lock()
    defer pe.mu.Unlock()
    
    pe.logger.Info("Processing event", 
        "event", event.Event, 
        "current_state", pe.stateMachine.current)
    
    // Update current time
    pe.context.CurrentTime = time.Now().UTC()
    
    // Find matching transition
    transition := pe.stateMachine.FindTransition(pe.stateMachine.current, event.Event)
    if transition == nil {
        pe.logger.Debug("No transition for event in current state",
            "event", event.Event,
            "state", pe.stateMachine.current)
        return
    }
    
    // Evaluate guard
    if transition.Guard != nil && !transition.Guard(pe.context) {
        pe.logger.Debug("Transition guard failed",
            "event", event.Event,
            "from", transition.FromState,
            "to", transition.ToState)
        return
    }
    
    // Cancel current state timeout
    pe.timeoutManager.CancelTimeout(pe.pass.ID, pe.stateMachine.current)
    
    // Execute transition actions
    for _, action := range transition.Actions {
        if err := action(pe.context); err != nil {
            pe.logger.Error("Transition action failed", 
                "error", err,
                "action", getFunctionName(action))
            
            // Handle error
            if err := pe.errorHandler.HandleError(pe.context, err); err != nil {
                pe.logger.Error("Error handling failed, aborting", "error", err)
                pe.transitionTo(StateFailed)
                return
            }
        }
    }
    
    // Update state
    oldState := pe.stateMachine.current
    pe.stateMachine.current = transition.ToState
    
    // Record state change
    change := StateChange{
        FromState:   oldState,
        ToState:     transition.ToState,
        Event:       event.Event,
        Timestamp:   pe.context.CurrentTime,
        TriggeredBy: event.RequestedBy,
    }
    pe.stateMachine.history = append(pe.stateMachine.history, change)
    
    // Publish state change event
    pe.publishStateChange(change)
    
    // Start timeout for new state
    pe.timeoutManager.StartTimeout(pe.pass.ID, transition.ToState)
    
    pe.logger.Info("State transition complete",
        "from", oldState,
        "to", transition.ToState,
        "event", event.Event)
}

// FindTransition finds a matching transition for state and event
func (sm *StateMachine) FindTransition(state State, event Event) *Transition {
    for i := range sm.transitions {
        t := &sm.transitions[i]
        
        // Check for exact state match or wildcard
        stateMatch := t.FromState == state || t.FromState == StateAny
        
        // For StateAny, check if current state is in active category
        if t.FromState == StateAny {
            stateMatch = isActiveState(state)
        }
        
        if stateMatch && t.Event == event {
            return t
        }
    }
    return nil
}

// SubmitEvent adds an event to the processing queue
func (pe *PassExecutor) SubmitEvent(event EventPayload) {
    select {
    case pe.eventChan <- event:
        // Event queued
    default:
        pe.logger.Warn("Event queue full, dropping event", "event", event.Event)
    }
}

// GetState returns the current state
func (pe *PassExecutor) GetState() State {
    pe.mu.RLock()
    defer pe.mu.RUnlock()
    return pe.stateMachine.current
}

// GetHistory returns state transition history
func (pe *PassExecutor) GetHistory() []StateChange {
    pe.mu.RLock()
    defer pe.mu.RUnlock()
    return pe.stateMachine.history
}
```

### 14.2 Event Publishing

```go
// event_publisher.go

package passexec

import (
    "encoding/json"
    "github.com/nats-io/nats.go"
)

// EventPublisher publishes pass execution events
type EventPublisher struct {
    nats      *nats.Conn
    jetStream nats.JetStreamContext
}

// PublishStateChange emits a state transition event
func (ep *EventPublisher) PublishStateChange(change StateChange, passID string) error {
    event := map[string]interface{}{
        "type":           "pass.state.changed",
        "pass_id":        passID,
        "previous_state": change.FromState,
        "current_state":  change.ToState,
        "event":          change.Event,
        "timestamp":      change.Timestamp.Format(time.RFC3339Nano),
        "triggered_by":   change.TriggeredBy,
    }
    
    data, err := json.Marshal(event)
    if err != nil {
        return err
    }
    
    subject := fmt.Sprintf("pass.%s.state", passID)
    _, err = ep.jetStream.Publish(subject, data)
    return err
}

// PublishMetricsUpdate emits metrics update
func (ep *EventPublisher) PublishMetricsUpdate(passID string, metrics *PassMetrics) error {
    event := map[string]interface{}{
        "type":      "pass.metrics.updated",
        "pass_id":   passID,
        "metrics":   metrics,
        "timestamp": time.Now().UTC().Format(time.RFC3339Nano),
    }
    
    data, _ := json.Marshal(event)
    subject := fmt.Sprintf("pass.%s.metrics", passID)
    _, err := ep.jetStream.Publish(subject, data)
    return err
}

// PublishAlert emits an alert for operator attention
func (ep *EventPublisher) PublishAlert(passID string, alert *OperatorAlert) error {
    event := map[string]interface{}{
        "type":        "pass.alert",
        "pass_id":     passID,
        "alert_type":  alert.Severity,
        "message":     alert.Message,
        "requires_ack": alert.RequiresAck,
        "timestamp":   alert.Time.Format(time.RFC3339Nano),
    }
    
    data, _ := json.Marshal(event)
    
    // Publish to both pass-specific and alerts topic
    ep.jetStream.Publish(fmt.Sprintf("pass.%s.alerts", passID), data)
    ep.jetStream.Publish("pass.alerts", data)
    
    return nil
}
```

### 15.3 Alerting Rules

```yaml
# prometheus-alerts-pass-execution.yaml

groups:
  - name: pass-execution
    rules:
      - alert: PassPreparationFailed
        expr: increase(pass_execution_state_transitions_total{to_state="PREP_FAILED"}[1h]) > 2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Multiple pass preparation failures"
          description: "{{ $value }} passes failed preparation in last hour"
          
      - alert: PassAcquisitionTimeout
        expr: pass_execution_acquisition_duration_seconds > 60
        for: 0m
        labels:
          severity: warning
        annotations:
          summary: "Pass acquisition taking too long"
          description: "Pass {{ $labels.pass_id }} acquisition exceeded 60s"
          
      - alert: PassAborted
        expr: increase(pass_execution_aborted_total[1h]) > 0
        for: 0m
        labels:
          severity: critical
        annotations:
          summary: "Pass was aborted"
          description: "Pass aborted at {{ $labels.station_id }}: {{ $labels.reason }}"
          
      - alert: LowPassSuccessRate
        expr: |
          sum(rate(pass_execution_completed_total{result="SUCCESS"}[24h])) /
          sum(rate(pass_execution_completed_total[24h])) < 0.9
        for: 1h
        labels:
          severity: warning
        annotations:
          summary: "Pass success rate below 90%"
          description: "24-hour pass success rate is {{ $value | humanizePercentage }}"
          
      - alert: NoPassesScheduled
        expr: pass_execution_passes_scheduled == 0
        for: 6h
        labels:
          severity: info
        annotations:
          summary: "No passes scheduled"
          description: "Station {{ $labels.station_id }} has no scheduled passes"
```

```yaml
      duration: 10m
      action: FAILED

    category: pre_pass
    timeout: null

    category: active
    timeout:
      duration: 60s
      action: conditional  # Check LOS proximity
      
  - name: TRACKING
    category: active
    timeout: null
    sub_states:
      - NOMINAL
      - DEGRADED
      - COMMANDING
      
  - name: CLOSING
    category: active
    timeout:
      duration: 2m
      action: REPORTING
      
  - name: REPORTING
    category: post_pass
    timeout:
      duration: 5m
      action: COMPLETED
      
  - name: COMPLETED
    category: terminal
    
  - name: FAILED
    category: terminal
    
  - name: CANCELLED
    category: terminal
    
  - name: ABORTED
    category: terminal

transitions:
  - from: SCHEDULED
    to: PREPARING
    event: PREP_TIME_REACHED
    guard: time_to_aos_within_lead
    
  - from: SCHEDULED

    event: CANCEL_REQUESTED
```

## Appendix B: Glossary

| Term | Definition |
|------|------------|
| **AOS** | Acquisition of Signal - when satellite rises above horizon |
| **LOS** | Loss of Signal - when satellite sets below horizon |
| **Guard** | Boolean condition for allowing transition |
| **Action** | Side effect executed during transition |
| **Terminal State** | Final state with no outgoing transitions |
| **Sub-state** | Nested state within a parent state |
| **Reacquisition** | Regaining signal lock after temporary loss |
