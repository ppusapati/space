## 2. E-EO-001: Imagery Catalog

> **Epic Goal:** Provide STAC-compliant imagery catalog with spatial/temporal search

### US-EO-001: Create Collection

**Epic:** E-EO-001  
**Use Case:** UC-EO-001  
**Priority:** Must Have  
**Story Points:** 5

**As a** data curator  
**I want to** create imagery collections  
**So that** images are organized by source and type

**Acceptance Criteria:**

```gherkin
Scenario: Create STAC collection
  Given I am an authenticated data curator
  When I create a collection:
    | Field       | Value                        |
    | id          | sentinel-2-l2a               |
    | title       | Sentinel-2 Level-2A          |
    | description | Atmospherically corrected    |
    | license     | CC-BY-4.0                    |
    | extent      | Global, 2015-present         |
  Then a STAC-compliant collection is created
  And it's available for item ingestion

Scenario: Collection with providers
  Given I am creating a collection
  When I add provider information:
    | Name | Role      | URL                    |
    | ESA  | producer  | https://esa.int        |
    | P9E  | host      | https://p9e.space      |
  Then providers are stored in collection metadata

Scenario: Collection extent
  Given I am defining collection extent
  When I specify:
    - Spatial: bbox or geometry
    - Temporal: start/end dates
  Then the extent is validated and stored
```

**Technical Notes:**
- STAC 1.0.0 specification compliance
- Support custom extensions (eo, sat, view)
- Store in PostgreSQL with PostGIS
- Auto-calculate extent from items

**Dependencies:** E-PLT-002 (Tenant)
